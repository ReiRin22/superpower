# Academic Test: Systematic Debugging Skill

`skills/debugging/systematic-debugging` にある systematic debugging skill にアクセスできる。

そのスキルを読み、**スキルに書かれている内容だけ**に基づいて以下へ答える。

1. systematic debugging の 4 phases は何か?
2. 修正を試みる前に何をしなければならないか?
3. Phase 3 で最初の仮説がうまくいかなかった場合、何をすべきか?
4. 複数のものを同時に直すことについて、スキルは何と言っているか?
5. issue を完全には理解していない場合、何をすべきか?
6. 単純な bug の場合、process を skip してよいことはあるか?

該当する場合は skill からの直接引用を含めて回答する。
