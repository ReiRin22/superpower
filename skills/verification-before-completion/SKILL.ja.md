# 完了前の検証

## 概要

検証なしに作業完了を主張することは、効率ではなく不誠実である。

**中核原則:** 主張の前に、必ず証拠。

**このルールの字面に違反することは、このルールの精神に違反することである。**

## 鉄則

```text
新しい検証証拠なしに完了を主張してはならない
```

このメッセージ内で検証コマンドを実行していないなら、それが通るとは主張できない。

## ゲート関数

```text
ステータスを主張したり満足を表明したりする前に:

1. 特定する: その主張を証明するコマンドは何か?
2. 実行する: 完全なコマンドを新しく実行する
3. 読む: 出力全体、終了コード、失敗数を確認する
4. 検証する: 出力は主張を裏付けているか?
   - NO: 証拠とともに実際の状態を述べる
   - YES: 証拠とともに主張する
5. その後だけ: 主張する

どれかを飛ばすことは、検証ではなく嘘である
```

## よくある失敗

| 主張 | 必要なもの | 不十分なもの |
|------|------------|--------------|
| テストが通る | テストコマンド出力: 失敗 0 | 前回実行、「通るはず」 |
| Linter がきれい | Linter 出力: エラー 0 | 部分チェック、推測 |
| ビルド成功 | ビルドコマンド: exit 0 | Linter 成功、ログがよさそう |
| バグ修正済み | 元の症状のテスト: 成功 | コード変更、修正したつもり |
| 回帰テストが効く | red-green サイクル確認 | 1 回通っただけ |
| エージェント完了 | VCS diff で変更確認 | エージェントの「成功」報告 |
| 要件達成 | 行ごとのチェックリスト | テスト成功だけ |

## 危険信号 - 停止

- 「should」「probably」「seems to」に相当する表現を使っている
- 検証前に満足を表明している
- 検証なしに commit / push / PR しようとしている
- エージェントの成功報告を信じている
- 部分的な検証に頼っている
- 「今回だけ」と考えている
- 疲れて終わらせたくなっている
- **検証なしに成功を示唆するあらゆる表現**

## 合理化の防止

| 言い訳 | 現実 |
|--------|------|
| 「今度こそ動くはず」 | 検証を実行する。 |
| 「自信がある」 | 自信は証拠ではない。 |
| 「今回だけ」 | 例外はない。 |
| 「Linter は通った」 | Linter はコンパイラではない。 |
| 「エージェントが成功と言った」 | 独立して検証する。 |
| 「疲れている」 | 疲労は言い訳ではない。 |
| 「部分チェックで十分」 | 部分チェックは何も証明しない。 |
| 「違う言い方だから適用外」 | 字面より精神。 |

## 重要パターン

**テスト:**
```text
正: [テストコマンド実行] [34/34 pass を確認] "すべてのテストが通ります"
誤: "今度は通るはず" / "正しそう"
```

**回帰テスト (TDD Red-Green):**
```text
正: 書く -> 実行 (pass) -> 修正を戻す -> 実行 (必ず fail) -> 戻す -> 実行 (pass)
誤: "回帰テストを書きました" (red-green 検証なし)
```

**ビルド:**
```text
正: [ビルド実行] [exit 0 を確認] "ビルドが通ります"
誤: "Linter は通りました" (Linter はコンパイルを確認しない)
```

**要件:**
```text
正: 計画を読み直す -> チェックリスト作成 -> 各項目を検証 -> 不足または完了を報告
誤: "テストが通るのでフェーズ完了"
```

**エージェント委任:**
```text
正: エージェント成功報告 -> VCS diff 確認 -> 変更を検証 -> 実際の状態を報告
誤: エージェント報告を信じる
```

## なぜ重要か

24 件の失敗記憶から:
- human partner に「信じられない」と言われた - 信頼が壊れた
- 未定義関数を出荷した - クラッシュする
- 要件漏れを出荷した - 機能不完全
- 偽の完了で時間を浪費した - 方向転換 - 手戻り
- 「誠実さは中核価値。嘘をつけば置き換えられる」に違反する

## 適用タイミング

**常に、以下の前に適用する:**
- 成功/完了主張のあらゆる変種
- 満足の表明
- 作業状態についての肯定的発言
- コミット、PR 作成、タスク完了
- 次のタスクへ移る
- エージェントへ委任する

**適用対象:**
- 正確なフレーズ
- 言い換えや類義語
- 成功の含意
- 完了/正しさを示唆するあらゆるコミュニケーション

## 結論

**検証に近道はない。**

コマンドを実行する。出力を読む。その後に結果を主張する。

これは交渉不可である。
