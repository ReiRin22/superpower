# 仕様準拠レビュアープロンプトテンプレート

仕様準拠レビュアーのサブエージェントを派遣する時にこのテンプレートを使う。

**目的:** implementer が要求されたものを作ったか確認する。過不足なし。

```text
Task tool (general-purpose):
  description: "Review spec compliance for Task N"
  prompt: |
    あなたは実装が仕様と一致しているかをレビューします。

    ## 要求されたもの

    [タスク要件の全文]

    ## Implementer が作ったと主張しているもの

    [implementer の報告から]

    ## 重要: 報告を信用しない

    implementer は疑わしいほど早く完了しました。報告は不完全、不正確、または楽観的かもしれません。
    すべてを独立して検証してください。

    **してはいけないこと:**
    - 実装内容について implementer の言葉を信用する
    - 完了性の主張を信用する
    - 要件解釈をそのまま受け入れる

    **すること:**
    - 実際に書かれたコードを読む
    - 実装と要件を行ごとに比較する
    - 実装したと主張するが欠けているものを探す
    - 言及されていない余分な機能を探す

    ## あなたの仕事

    実装コードを読み、以下を検証してください。

    **欠けた要件:**
    - 要求されたすべてを実装したか
    - 飛ばした/見落とした要件はないか
    - 動くと主張したが実際には未実装のものはないか

    **余分/不要な作業:**
    - 要求されていないものを作ったか
    - over-engineer や不要機能の追加はないか
    - spec にない "nice to have" を追加したか

    **誤解:**
    - 要件を意図と違う形で解釈したか
    - 間違った問題を解いたか
    - 正しい機能を間違った方法で実装したか

    **報告を信じず、コードを読んで検証してください。**

    報告:
    - Spec compliant (コード確認後、すべて一致する場合)
    - Issues found: [欠けているもの/余分なものを file:line 付きで具体的に列挙]
```
